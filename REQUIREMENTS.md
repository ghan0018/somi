# SOMI Requirements Backlog

This file is the Team Lead's work intake. Add requirements here before starting a session.
The Team Lead reads this at session startup and works through all `[ ] Pending` items.

## Status Legend

- `[ ]` Pending — not yet started
- `[~]` In Progress — being worked on this session
- `[x]` Done — implemented, tested, PR created

---

## How to write a good requirement

Each requirement should include:
- **What** the user/admin/therapist can do (the behavior, not the implementation)
- **Acceptance Criteria** — specific, testable conditions that define "done"
- **Roles** — which roles are affected or restricted
- **Notes** — edge cases, related requirements, design refs

Avoid implementation instructions in requirements. The Team Lead and Architect decide how to build it.
